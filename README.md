## Что внутри репозитория

- `9.1` - Alembic, модель `Product`, миграции `create table`, `seed`, `add description`
- `10.1` - пользовательские исключения FastAPI, обработчики, модель ошибки, тесты
- `10.2` - валидация JSON и кастомная обработка ошибок валидации
- `11.1` - синхронные модульные тесты для 3 endpoint
- `11.2` - асинхронные тесты с `pytest-asyncio`, `httpx.AsyncClient`, `ASGITransport`, `Faker`

## Как скачать проект

### Вариант 1: через git clone

```bash
git clone https://github.com/Rithmer/fapi-kr4.git
cd project4
```

### Вариант 2: скачать ZIP

1. Откройте страницу репозитория на GitHub.
2. Нажмите `Code` -> `Download ZIP`.
3. Распакуйте архив и откройте папку проекта в VS Code.

## Настройка окружения

Из корня проекта:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Проверка, что зависимости установлены:

```bash
python -m pytest --version
```

## Переменные окружения (.env)

Скопируйте шаблон:

```bash
cp .env.example .env
```

Пример содержимого:

```env
DATABASE_URL=sqlite:///./app.db
```

## Как запускать задания

Папки называются `9.1`, `10.1`, `10.2`, `11.1`, `11.2`, поэтому запуск и тестирование нужно выполнять внутри нужной папки.

Общий формат запуска приложения:

```bash
uvicorn app.main:app --reload
```

Общий формат запуска тестов:

```bash
python -m pytest -q
```

## Как тестировать каждое задание

### 9.1 - Alembic migrations

```bash
cd 9.1
alembic upgrade head
```

Проверка результата:

1. Таблица `products` должна создаться.
2. В таблице должны быть 2 записи.
3. В таблице должен быть столбец `description` с `NOT NULL`.

Дополнительная проверка:

```bash
uvicorn app.main:app --reload
```

Откройте `GET /products` в Swagger или через браузер.

### 10.1 - Custom errors

```bash
cd 10.1
uvicorn app.main:app --reload
```

Проверка:

1. `GET /exception-a` должен вернуть `418`.
2. `GET /exception-b/123` должен вернуть `404`.
3. Ответы должны содержать поля `error` и `code`.
4. `python -m pytest -q` должен пройти без ошибок.

### 10.2 - Request validation

```bash
cd 10.2
uvicorn app.main:app --reload
```

Проверка:

1. Отправьте валидный JSON на `POST /validate-user` и проверьте `200`.
2. Отправьте невалидный JSON с ошибками в `age`, `email` или `password`.
3. В ответе должен быть `422` и единый формат ошибки.
4. `python -m pytest -q` должен пройти.

### 11.1 - Sync tests

```bash
cd 11.1
python -m pytest -q
```

Тесты должны проверить:

1. создание пользователя `201`;
2. получение пользователя `200`;
3. получение несуществующего пользователя `404`;
4. удаление пользователя `204`;
5. повторное удаление `404`.

### 11.2 - Async tests

```bash
cd 11.2
python -m pytest -q
```

Тесты должны подтвердить:

1. использование `pytest-asyncio`;
2. вызовы через `httpx.AsyncClient` + `ASGITransport`;
3. генерацию данных через `Faker`;
4. сценарии `201`, `200`, `404`, `204`, `404`;
5. изоляцию состояния между тестами.
